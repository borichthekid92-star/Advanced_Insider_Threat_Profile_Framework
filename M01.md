## M01: Disgruntled Employee 

### Description – This profile details an insider who, driven by personal grievance against their employer, acts with deliberate malicious intent from within a position of legitimate access. The disgruntled employee does not exploit a technical vulnerability — they exploit trust. Their access is authorized, their presence is expected, and their knowledge of internal systems, processes, and people makes them uniquely dangerous. The harm they cause is rarely impulsive; it is the product of accumulated resentment that has crossed a threshold, converting a once-productive employee into an active threat.

Psychology – The psychological trajectory of the disgruntled insider is rarely sudden. It develops along a continuum — from perceived injustice to emotional withdrawal, and eventually to a decision to act. Common catalysts include denied promotions, termination notices, disciplinary actions, interpersonal conflicts with management, feelings of being undervalued, or a belief that the organization has treated them unfairly. Over time, these feelings compound into resentment, helplessness, and in some cases a desire for retaliation or revenge.

Example – A senior user working for ducks co was passed on a promotion 3 months ago. The user has been vocal about how upset they are and begin putting less time into work projects. Next they call out sick during an important zoom call. The user then begins to think of ways to steal their coworkers password. They tell their coworker that their manager needed a copy of their current password to check something. With intentions to sell the password to a telegram connection they save the pw and open notepad and paste the pw.

### Preparation: 
The actor may be still contemplating the malicious plan and subtly putting together pieces, or they may be doing things with subconscious want of getting caught.  
Detection: Large file compressions or decompressions, Large volume of file movement, files being moved in personal folders, copy and pasting from sensitive files into new documents, high amount of files being pulled from cloud storage over time, large volume of copy and pasting, copying from sensitive files, renaming sensitive files, new local share drive, new usb device, researching technical commands, researching behavioral anomalous content, researching unusual tools not related to job on github or search 
Mitigations: Audit logging on sensitive files.  
### Access Abuse:
The actor uses their current credentials to conduct recon, move around files, conduct research, elevate privileges or steal credentials. Users with high privileges will be using their higher permissions more than usual. 
Detection: User leverages current permissions to access large amounts of files, or sensitive files for potential recon. User searches for passwords. User accessing new remote hosts with creds. User leverages current position to attempt to elevate privileges or obtain other credentials. User stages payload files with current access. 
### Reconnaissance 
The actor searches for file shares, user information, system vulnerabilities, sensitive documents, or accessible network resources to build a map of what they can reach and what would cause the most damage or yield the most valuable data for sale or leverage.
Detection
Net share / net use enumeration commands executed. LDAP or Active Directory queries for user, group, or computer listings. Bulk browsing of file shares across multiple network paths. Accessing HR, Finance, or Legal document stores outside normal job role. Querying internal wikis or SharePoint sites outside assigned team spaces. Running port scan or network discovery tools (nmap, Advanced IP Scanner, etc.). Accessing org chart tools or employee directories in bulk. Searching internal ticketing systems for vulnerability disclosures or patch gaps
Mitigations
Monitor and alert on AD enumeration queries (Event IDs 4661, 4662)
Restrict access to HR, Finance, and Legal repositories by role group
Alert on execution of network scanning binaries on non-IT endpoints
Honeypot file shares and documents — any access triggers immediate alert
SharePoint and Confluence access auditing with role-deviation alerting
### Collection
The actor begins actively gathering targeted data — proprietary documents, credentials, customer records, source code, or financial data — often in bulk. At this stage the actor may still be emotional and less disciplined, resulting in noisier activity compared to more sophisticated insider types. Volume and velocity anomalies are most detectable here.
Detection
Sudden spike in file access volume compared to personal baseline
Accessing and opening files well outside normal job function
User moving scripts and binaries from external source
Downloading large numbers of documents from cloud repositories
Email forwarding rules created pointing to external personal accounts
Aggregating files from multiple sources into a single staging folder
Screenshots taken immediately after opening sensitive documents
Clipboard activity capturing sensitive file content
Users leveraging cli tools to aggregate data
Source code repository mass clone or pull requests
Accessing backup directories or archival storage
User’s creating backup folder and moving mass files there
Mitigations
DLP rules on volume-based file access anomalies (e.g., 3x peer baseline)
Alert on new external email forwarding rules in Exchange / M365
Print server logging with sensitive document print alerts
Cloud repository download volume thresholds with automated holds
Screenshot and clipboard monitoring via endpoint DLP agent
File staging folder detection — multiple-source aggregation into one path
### Exfiltration → 
Data is transferred to the external party using channels that circumvent corporate network monitoring. Some actors may use regular channels. The actor prefers off-network transfers, encrypted dead drops, or physical carry-out to ensure no network-level telemetry is generated inside the corporate perimeter.
Techniques
DNS exfiltration, personal NAS storage, temporary http server, self email on personal account, USB transfer, Upload to personal cloud storage via personal hotspot, USB carry-out from facility, Bluetooth/AirDrop transfer to personal device, Encrypted email to external personal account, Dead drop to cloud storage shared folder, Transfer during travel to evade network controls, Steganographic file posted to public social platform
Misc: rsync, stfp, scp, nslookup, powershell, smbd, wget, AWS CLI S3 sync, Curl (HTTP POST), Robocopy, Xcopy, TeraCopy, FastCopy, Socat, DNScat2, Pastebin, WeTransfer, SendAnywhere, Mega, Dropbox
